# World Cup Predictor

End-to-end probabilistic prediction for the 2026 FIFA World Cup. The project:

- predicts expected goals, win/draw/loss, likely score, over 2.5, and BTTS;
- updates team attack/defense state after every completed match;
- uses dated FIFA rankings and annual player-talent snapshots, plus availability,
  expected minutes, position, depth, star power, age, and chemistry;
- supports historical xG and market closing odds when available;
- simulates all 104 matches and reports advancement/championship probabilities;
- validates with a strict chronological split and calibration-focused metrics.

It models goals rather than directly classifying the winner. This keeps exact
scores, W/D/L, totals, and tournament simulations internally consistent.

## Quick Start

Requires Python 3.11 or newer.

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -e ".[dev]"

worldcup download-data
worldcup train
worldcup predict
worldcup simulate
worldcup log        # review the archived forecast history
```

Generated files:

| File | Contents |
|---|---|
| `PREDICTIONS.md` | clean human-readable summary, regenerated on every predict/simulate |
| `artifacts/model.joblib` | symmetric goal model, dispersion, calibration, and Dixon-Coles parameter |
| `artifacts/metrics.json` | temporal holdout metrics and active feature list |
| `artifacts/team_state.json` | current Elo plus attack/defense means and variances |
| `outputs/match_predictions.csv` | predictions for unplayed group matches |
| `outputs/tournament_probabilities.csv` | advancement and title probabilities |
| `predictions_log/<date>/` | archived forecast checkpoints (see [Forecast logbook](#forecast-logbook)) |

All paths and model settings are in `config.toml`.

## Data

### Historical matches

`worldcup download-data` downloads the CC0
[International football results dataset](https://github.com/martj42/international_results)
to `data/raw/results.csv`. Unplayed rows are automatically excluded.

### 2026 schedule

The same command creates `data/input/schedule_2026.csv` from the
[openfootball World Cup data](https://github.com/openfootball/worldcup). It
contains the 72 group matches and the official 32-match knockout structure.

### FIFA rankings

`data/input/rankings.csv` ships with 70,000+ dated snapshots from August 1993
through March 2026:

```csv
date,team,rank,points
2026-04-01,Example Team,12,1684.20
```

Regenerate it with `python scripts/build_rankings.py`. Source URLs and coverage
notes are recorded in `data/input/rankings_sources.txt`.

### Player talent and chemistry

`data/input/squads.csv` is optional:

```csv
as_of,team,player,club,position,talent,talent_se,exp_minutes,available,age,is_starter
2026-06-01,Example Team,Player One,Example FC,FWD,88,2,0.90,true,27,true
```

| Column | Meaning |
|---|---|
| `as_of` | date this squad/lineup snapshot became known |
| `team` | national team name |
| `player` | player name or stable identifier |
| `club` | current club, used for same-club chemistry |
| `position` | `GK`, `DEF`, `MID`, or `FWD`; common EA positions are normalized |
| `talent` | source rating, such as EA FC overall or transformed market value |
| `talent_se` | uncertainty in the talent value; defaults to 3 when blank |
| `exp_minutes` | expected share of available minutes from 0 to 1 |
| `available` | false for injury or suspension |
| `age` | age on the snapshot date |
| `is_starter` | expected or confirmed starter |

The practical free starting point is the public
[EA SPORTS FC ratings database](https://www.ea.com/games/ea-sports-fc/ratings).
Transfermarkt market values can also be folded in to cover players EA does not
license: `scripts/download_transfermarkt.py` pulls them from a **self-hosted**
[felipeall/transfermarkt-api](https://github.com/felipeall/transfermarkt-api)
instance — there is no direct, fragile scraping of `transfermarkt.com` — and
`scripts/apply_transfermarkt.py` calibrates those values onto the EA talent
scale (`talent ~ log10(value) + age`) to replace floor-imputed players.

The shipped file combines annual FIFA/EA talent proxies from 2014 onward with
the actual June 2026 squad snapshot. Regenerate the historical portion with
`python scripts/build_historical_squads.py`. Source URLs are recorded in
`data/input/squads_sources.txt`.

The current squad was generated by `scripts/download_ea_ratings.py` and
`scripts/build_squads.py`. Its imputed or non-exact matches remain listed in
`data/input/squads_match_report.txt`.

Use the same talent scale across all teams in a snapshot. Historical snapshots
allow the model to learn player-feature coefficients. Manual
talent/ranking/chemistry fallback priors are disabled by default.

### xG and market odds

`data/input/match_features.csv` includes StatsBomb xG for all 128 matches in
the 2018 and 2022 World Cups. Regenerate it with
`python scripts/build_match_features.py`.

Additional rows may include decimal closing odds:

```csv
date,home_team,away_team,home_xg,away_xg,home_odds,draw_odds,away_odds
```

Odds are converted to overround-free implied probabilities. No odds ship with
the project because a complete, clearly licensed international closing-line
archive was not available.

## Model

### Leakage-safe match features

For every historical match, features are computed before its result is applied:

- recency-weighted Elo;
- recent points, goals, and xG where available;
- rest, travel, altitude, climate, venue/host advantage, and match importance;
- knockout-stage and confederation context;
- Bayesian attack/defense means and uncertainty;
- latest available FIFA ranking snapshot;
- squad talent, star power, depth, availability, age, and chemistry.

Old matches receive exponentially lower weight. Friendlies also receive lower
weight than qualifiers and major tournament matches.

### Player roll-up

Player contribution is weighted by expected minutes and availability. Attack
weights favor forwards and midfielders; defense weights favor defenders and
midfielders. The likely goalkeeper is handled separately. Same-club starter
pairs produce a small chemistry feature rather than dominating team strength.

### Goal model

One Poisson-loss histogram gradient-boosted model is trained on every match from
both team perspectives. This gives exact label symmetry and twice the training
rows. Rates are converted into an `11 x 11` negative-binomial score grid with
learned dispersion. A fitted Dixon-Coles parameter corrects the four low-score
cells: `0-0`, `0-1`, `1-0`, and `1-1`. An optional temperature term can
recalibrate the 1X2 regions of that grid; by default it is fitted on the model's
own validation holdout during `worldcup train`, because a temperature fitted on
a single held-out World Cup does not generalize to the next one (see
[Calibration](#calibration)).

The grid produces:

- `P(home win)`, `P(draw)`, and `P(away win)`;
- most likely score;
- expected goals;
- over/under 2.5;
- both teams to score.

### Live updating

`TeamState` contains Elo and separate attack/defense means and variances. After
each result, surprise relative to expected scoring updates those latent states
with a conservative Kalman-style gain. Uncertain teams move more; established
teams move less. Variance shrinks after observation and drifts upward between
matches.

Add a completed result:

```bash
worldcup update \
  --date 2026-06-11 \
  --home Mexico \
  --away "South Africa" \
  --home-score 2 \
  --away-score 0 \
  --no-neutral \
  --country Mexico
```

This updates `results.csv` and `team_state.json` without retraining the global
model. Use `--retrain` when enough new data has accumulated to justify a refit.

> **Keeping predictions current once the tournament is underway.** With the
> default `cutoff_date` and `include_tournament = false`, the simulator treats
> every World Cup match as unplayed and keeps re-simulating games that have
> already finished. After recording results with `worldcup update`, set
> `include_tournament = true` in `config.toml` so completed group/knockout
> games are pinned as observed outcomes and only the remaining matches are
> sampled.

For a knockout match tied after extra time, provide the advancing team:

```bash
worldcup update ... --home-score 1 --away-score 1 --winner Mexico
```

Then rerun `worldcup predict` and `worldcup simulate`.

### Readable summary

`worldcup predict` and `worldcup simulate` also regenerate `PREDICTIONS.md`, a
single clean file with the title/advancement table, every group-stage match
prediction, and a model snapshot. It is overwritten on every run, so after the
usual `train` → `predict` → `simulate` flow it always shows the latest forecast
on the latest data. Regenerate it on demand with `worldcup report`.

### Forecast logbook

Every `worldcup predict` and `worldcup simulate` run archives a snapshot under
`predictions_log/`, so the forecast is never silently overwritten as the
tournament unfolds. Snapshots are keyed by the latest played-result date (the
"data through" point), so each matchday you fold in with `worldcup update`
creates a new, comparable checkpoint while the earlier ones are preserved:

```text
predictions_log/
├── index.jsonl                 # one line per recorded run
├── 2026-06-10/                 # pre-tournament forecast
│   ├── match_predictions.csv
│   ├── tournament_probabilities.csv
│   └── manifest.json           # data-through, model version, git commit, top odds
└── 2026-06-18/                 # after matchday 1 results were added
    └── ...
```

Review the timeline with:

```bash
worldcup log
```

This directory is committed (unlike `outputs/`), so the evolution of the model's
title odds across the tournament becomes a permanent, auditable record.

### Tournament simulation

Simulation uses actual completed World Cup results as fixed observations and
samples only remaining matches. It implements group standings, head-to-head
tiebreaking, randomized resolution after unavailable fair-play criteria, the
eight best third-place teams, the 48-team bracket, extra time, and penalties.

Each match is sampled from the fitted dispersion-aware Dixon-Coles grid, so
match probabilities and Monte Carlo use the same joint goal distribution.
Each simulated tournament also draws one latent strength offset per team, held fixed
across all of that team's matches in that run and scaled by the team's
posterior uncertainty. This keeps a team that is randomly stronger than its
point estimate consistently stronger all tournament, so advancement and title
spreads are realistically wide rather than artificially concentrated by
independent per-match noise. The spread scale was selected on 2022 advancement
Brier scores. The shootout Elo weight was fitted on 415 historical
international shootouts.

Rates for feasible bracket matchups are predicted in batches before Monte Carlo,
so thousands of runs remain practical.

```bash
worldcup simulate --runs 10000 --seed 42
```

## Evaluation

`worldcup train` uses a chronological 80/20 split. Feature generation itself is
walk-forward: no match result enters its own or an earlier match's features.
Alongside the 1X2 metrics, `metrics.json` now also reports Brier score and
calibration error for the over/under 2.5 and both-teams-to-score markets, so the
secondary outputs are checked rather than assumed. The Elo baseline is a
multinomial logistic on the Elo gap and venue advantage, fit on the same
recency/importance weights (a fairer comparison than the old fixed-goal mapping).

The current training run uses 25,343 completed matches from January 1, 2000
through June 10, 2026:

| Metric | Model | Elo baseline |
|---|---:|---:|
| Ranked Probability Score | 0.1682 | 0.1764 |
| Outcome log loss | 0.8616 | 0.8926 |
| Brier score | 0.5062 | - |
| Calibration error | 0.0115 | - |
| Home-goal MAE | 0.9941 | - |
| Away-goal MAE | 0.8307 | - |

Lower is better. These results are a temporal holdout, not evidence that the
model will beat bookmaker closing odds. The stricter whole-2022-tournament
backtest remains worse: outcome log loss `1.0241` and RPS `0.2099`. Treat that
tournament result as the more honest estimate of World Cup performance.

### Calibration

The 1X2 probabilities can be tempered by a single scalar. `scripts/calibrate_tournaments.py`
fits that scalar on the held-out 2018 World Cup and validates it on the 2022
World Cup, which the fit never sees. The finding: a temperature fitted on one
tournament (~0.88) is marginally *worse* on the next, so the apparent
overconfidence is mostly tournament-to-tournament noise rather than a stable
bias. For that reason no temperature is pinned in `config.toml` by default;
`worldcup train` instead fits it on its own ~5,000-match validation holdout,
which is far more stable. Pin `[calibration] temperature` only if you have a
reason to force a specific value (it is then applied in backtests too).

### Backtesting

A single split can be noisy. `worldcup backtest` trains on everything before
each fold boundary and evaluates on the next window, reusing the same
walk-forward features so a test fold never sees its own or later results:

```bash
worldcup backtest --cutoffs 2022-01-01,2023-01-01,2024-01-01,2025-01-01
```

It writes per-fold metrics to `outputs/backtest.csv` and prints the mean across
folds. With no `--cutoffs`, it defaults to January 1 of each of the last four
seasons.

The model/state/Elo search and simulation calibrations are reproducible:

```bash
python scripts/tune_parameters.py 8
python scripts/calibrate_tournaments.py
python scripts/calibrate_shootouts.py
python scripts/calibrate_team_strength.py 750
```

For a whole-tournament out-of-sample check, `scripts/backtest_2022.py` trains
only on pre-2022 matches, scores the 64 actual 2022 World Cup results, and
simulates the full 2022 bracket (`data/input/schedule_2022.csv`) to compare
advancement and title probabilities against what happened. The simulator derives
its structure from the schedule, so it runs the 2022 32-team format and the 2026
48-team format from the same code.

Run tests with:

```bash
pytest -q
```

## Limits

- Football outcomes remain noisy; calibration is more realistic than expecting
  a correct bracket.
- Historical squad rows are top-player proxies from game ratings, not confirmed
  lineups for every past match.
- Historical xG is limited to the 2018 and 2022 World Cups.
- No market closing odds ship with the project.
- Chemistry is correlation-heavy and deliberately capped.
- Confirmed lineups improve predictions; add a new dated squad snapshot when
  starters, injuries, or suspensions change.
- Penalty shootouts need `--winner` because a tied score alone does not identify
  the advancing team.
